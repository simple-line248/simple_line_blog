作为一名长期奋斗在一线框架设计与实现的前端开发人员，我对“美感”二字常常怀抱敬畏。技术世界通常是冰冷的，而视觉世界则是极其感性的。如何在理性的数据节点和结构当中，剥离并传现出“有机且有呼吸感的 UI”，一直是我们不断探索的课题。在这个长文测试页面中，我们将一起进行一场从微观 UI 到宏观架构的探索。

## 第一章：关于容器包裹感的可怕执念

无论文字是在屏幕的绝对中间，还是像古典杂志一样左侧靠齐留出海量右半区，其本质都是一种“容器设定”。这就好像水只能以杯子的形状示人一样，内容总是被布局锁死在一个既定认知框架内。

当文章的字数不够支撑起这种大型左右排布的骨架时，界面看起来仿佛丢失了一大块内存。

### 1.1 纸张的物理隐喻
人们对物理材质总有着某种执念。通过光影渲染出 `box-shadow` 和带有厚度的卡片背景（也就是此时此刻你看到的这个极其柔和的白色圆角长框），可以很好地欺骗大脑：即使这是一篇字数只有一百来字极其短薄的文章，大脑也会认为“它放在了一张大 A4 纸上”。

> 物理隐式的设计哲学，本质上是对人类几千年借助于纸媒介读取习惯的温柔继承。如果你去翻阅上世纪 90 年代的学术杂志，你会发现其字体排列、边距设定等，几乎都可以在今天的 CSS Grid/Flex 等实现里找到镜像。 

### 1.2 边距的数学：大屏幕上的小视口
通常由于宽屏时代的普及，人们越来越沉迷于“充满它”。其实这是一种误解——阅读是极其消耗眼球肌肉扫描能力的。单行文字越长，读者找下一行开头的成本就越高。所以我们将 `.reading-wrapper` 在中间死死掐在了最大 1000px 以内。

## 第二章：深入 Typography (排版学)

让我们继续写一些文字往下撑起这个巨大页面。现在你要盯着看右侧目录：随着文章高度的不断延伸，由于设置了 `position: sticky`，右侧目录应当安静地躺在那里充当向导。

### 2.1 字体设定与平滑度

让我们看一段简单的样式设置例子，在这个例子里我们应用了 `-webkit-font-smoothing: antialiased;` 技术，使 macOS 设备上的字体边缘消除了部分刺眼的锯齿。由于我们目前设定了暖色卡片效果，文字在卡片上的对比度实际上变温和了，没有在纯白背景下那么刺眼。

```css
html, body {
    margin: 0;
    padding: 0;
    background-color: var(--bg-color);
    color: var(--text-color);
    font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto...;
}
```

### 2.2 无序列表的设计实验
看看我们在排版中是处理这种短文本项的：

- 第一个小知识点：列表也需要一定的边距喘息。
- 第二个小知识点：通过 `line-height` 和 `margin-bottom` 提供节奏感。
- 第三个小知识点：在赭石红加持下，重点元素的吸引力将被微观上放大。
